Part 1: Lists — Creating, Accessing & Slicing

What is a List?

A list is an ordered, mutable collection of items enclosed in square brackets []

# Creating lists
fruits = ["apple", "banana", "cherry"]
scores = [85, 92, 78, 95, 88]
mixed = ["Sokha", 20, 3.75, True]   # Can mix types!
empty = []                            # Empty list

print(fruits)    # ['apple', 'banana', 'cherry']
print(scores)    # [85, 92, 78, 95, 88]
print(len(fruits))  # 3
print(len(scores))  # 5

💡 Key Facts About Lists

  • Ordered — items stay in the order you put them
  • Mutable — you CAN change items after creation (unlike strings!)
  • Allow duplicates — [1, 2, 2, 3] is perfectly valid
  • Can mix types — but usually you keep the same type

Accessing List Items (Indexing)

Just like strings, each item has a position (index) starting from 0:

fruits = ["apple", "banana", "cherry", "durian", "elderberry"]
#          0         1         2         3          4
#         -5        -4        -3        -2         -1

print(fruits[0])     # "apple"     (first item)
print(fruits[2])     # "cherry"    (third item)
print(fruits[-1])    # "elderberry" (last item)
print(fruits[-2])    # "durian"    (second from last)
# ❌ What happens if you go out of range?
print(fruits[10])    # IndexError: list index out of range

🎯 Same Rules as Strings!

Index starts at 0. Last item is list[-1]. Going out of range causes an IndexError.

List Slicing

fruits = ["apple", "banana", "cherry", "durian", "elderberry"]

# Slicing syntax: list[start:stop]
print(fruits[1:4])    # ['banana', 'cherry', 'durian']
print(fruits[:3])     # ['apple', 'banana', 'cherry']
print(fruits[2:])     # ['cherry', 'durian', 'elderberry']
print(fruits[-3:])    # ['cherry', 'durian', 'elderberry']

# With step
print(fruits[::2])    # ['apple', 'cherry', 'elderberry']
print(fruits[::-1])   # ['elderberry', 'durian', 'cherry', 'banana', 'apple']

💡 Slicing Rule

Same as strings! list[start:stop] gives items from start up to but NOT including stop.

Lists are Mutable! (Unlike Strings)

Mutable means you CAN change items after creating the list

fruits = ["apple", "banana", "cherry"]

# ✅ Change an item directly
fruits[1] = "blueberry"
print(fruits)    # ['apple', 'blueberry', 'cherry']

# ✅ Change a slice
fruits[0:2] = ["avocado", "blackberry"]
print(fruits)    # ['avocado', 'blackberry', 'cherry']

⚠️ Remember the Difference!

Strings are immutable → name[0] = "J" gives an ERROR

Lists are mutable → fruits[0] = "avocado" works perfectly!

Part 2: List Methods — Adding, Removing & Sorting

Adding Items to a List

fruits = ["apple", "banana"]

# append() — add ONE item to the END
fruits.append("cherry")
print(fruits)    # ['apple', 'banana', 'cherry']

# insert() — add ONE item at a SPECIFIC position
fruits.insert(1, "avocado")
print(fruits)    # ['apple', 'avocado', 'banana', 'cherry']

# extend() — add MULTIPLE items to the END
fruits.extend(["durian", "elderberry"])
print(fruits)    # ['apple', 'avocado', 'banana', 'cherry', 'durian', 'elderberry']

🎯 append() vs extend()

append() adds one item (even if it’s a list — it adds the whole list as one item!)

extend() adds each item from another list individually.

append() vs extend() — Be Careful!

list1 = [1, 2, 3]
list2 = [1, 2, 3]

# append a list → adds the LIST as one item
list1.append([4, 5])
print(list1)    # [1, 2, 3, [4, 5]]  ← nested list!

# extend with a list → adds EACH item
list2.extend([4, 5])
print(list2)    # [1, 2, 3, 4, 5]    ← flat list!
# You can also use + to combine lists (creates a NEW list)
a = [1, 2, 3]
b = [4, 5, 6]
c = a + b
print(c)    # [1, 2, 3, 4, 5, 6]
print(a)    # [1, 2, 3]  ← original unchanged!

Removing Items from a List

fruits = ["apple", "banana", "cherry", "banana", "durian"]

# remove() — remove the FIRST occurrence of a value
fruits.remove("banana")
print(fruits)    # ['apple', 'cherry', 'banana', 'durian']

# pop() — remove and RETURN item at a position (default: last)
last = fruits.pop()
print(last)      # "durian"
print(fruits)    # ['apple', 'cherry', 'banana']

second = fruits.pop(1)
print(second)    # "cherry"
print(fruits)    # ['apple', 'banana']

# del — delete by index
del fruits[0]
print(fruits)    # ['banana']

# clear() — remove ALL items
fruits.clear()
print(fruits)    # []

⚠️ Common Error

remove() raises a ValueError if the item is not in the list! Check first with if item in list:.

Sorting Lists

numbers = [42, 15, 88, 3, 67, 23]

# sort() — sorts IN PLACE (changes the original list)
numbers.sort()
print(numbers)    # [3, 15, 23, 42, 67, 88]

# Sort in reverse order
numbers.sort(reverse=True)
print(numbers)    # [88, 67, 42, 23, 15, 3]

# sorted() — returns a NEW sorted list (original unchanged)
original = [42, 15, 88, 3, 67]
new_sorted = sorted(original)
print(original)     # [42, 15, 88, 3, 67]  ← unchanged!
print(new_sorted)   # [3, 15, 42, 67, 88]
# Sort strings alphabetically
names = ["Chenda", "Sokha", "Ary", "Bopha", "Dara"]
names.sort()
print(names)    # ['Ary', 'Bopha', 'Chenda', 'Dara', 'Sokha']

# reverse() — reverses the order (NOT sorting!)
names.reverse()
print(names)    # ['Sokha', 'Dara', 'Chenda', 'Bopha', 'Ary']

Other Useful List Operations

numbers = [10, 20, 30, 20, 40, 20]

# count() — how many times does a value appear?
print(numbers.count(20))     # 3

# index() — what position is the FIRST occurrence?
print(numbers.index(30))     # 2

# min(), max(), sum() — built-in functions
print(min(numbers))    # 10
print(max(numbers))    # 40
print(sum(numbers))    # 140

# Average
average = sum(numbers) / len(numbers)
print(f"Average: {average:.1f}")   # Average: 23.3

# copy() — make a copy of the list
backup = numbers.copy()
backup[0] = 999
print(numbers)    # [10, 20, 30, 20, 40, 20]  ← original safe!
print(backup)     # [999, 20, 30, 20, 40, 20]

Summary Table — List Methods

Method What It Does Example
append(x) Add x to end [1,2].append(3) → [1,2,3]
insert(i, x) Add x at position i [1,3].insert(1, 2) → [1,2,3]
extend(list) Add all items from another list [1].extend([2,3]) → [1,2,3]
remove(x) Remove first x [1,2,2].remove(2) → [1,2]
pop(i) Remove & return item at i [1,2,3].pop(1) → returns 2
clear() Remove all items [1,2,3].clear() → []
sort() Sort in place [3,1,2].sort() → [1,2,3]
reverse() Reverse in place [1,2,3].reverse() → [3,2,1]
count(x) Count occurrences of x [1,2,2].count(2) → 2
index(x) Find position of first x [10,20,30].index(20) → 1
copy() Make a shallow copy backup = nums.copy()

Part 3: Looping Through Lists

for Loop with Lists

fruits = ["apple", "banana", "cherry", "durian"]

# Simple loop — get each item
for fruit in fruits:
    print(f"I like {fruit}!")

# Output:
# I like apple!
# I like banana!
# I like cherry!
# I like durian!
# Loop with numbers
scores = [85, 92, 78, 95, 88]

total = 0
for score in scores:
    total += score

average = total / len(scores)
print(f"Average: {average:.1f}")    # Average: 87.6

enumerate() — Get Index AND Value

fruits = ["apple", "banana", "cherry", "durian"]

# Without enumerate — just values
for fruit in fruits:
    print(fruit)

# With enumerate — index AND value
for index, fruit in enumerate(fruits):
    print(f"{index}: {fruit}")
# 0: apple
# 1: banana
# 2: cherry
# 3: durian

# Start counting from 1 (for display)
for num, fruit in enumerate(fruits, 1):
    print(f"{num}. {fruit}")
# 1. apple
# 2. banana
# 3. cherry
# 4. durian

💡 When to Use enumerate()

Use it whenever you need both the position and the value. It’s cleaner than using range(len(list)).

range() with Lists

fruits = ["apple", "banana", "cherry", "durian"]

# Using range(len()) — less Pythonic but sometimes useful
for i in range(len(fruits)):
    print(f"Index {i}: {fruits[i]}")

# Useful when you need to MODIFY items
scores = [85, 92, 78, 95, 88]

# Add 5 bonus points to every score
for i in range(len(scores)):
    scores[i] += 5

print(scores)    # [90, 97, 83, 100, 93]

Building Lists with Loops

# Create a list of squares
squares = []
for i in range(1, 11):
    squares.append(i ** 2)
print(squares)    # [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

# Filter: keep only passing scores
all_scores = [85, 42, 92, 55, 78, 38, 95, 61]
passing = []
for score in all_scores:
    if score >= 60:
        passing.append(score)
print(f"Passing: {passing}")    # [85, 92, 78, 95, 61]
# List comprehension — a shortcut! (same result)
squares = [i ** 2 for i in range(1, 11)]
print(squares)    # [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

passing = [s for s in all_scores if s >= 60]
print(passing)    # [85, 92, 78, 95, 61]

💡 List Comprehension

[expression for item in list] — a compact way to create lists.

[expression for item in list if condition] — with filtering.

It does the same thing as a loop + append, just shorter!

Part 4: Tuples — Immutable Sequences

What is a Tuple?

A tuple is an ordered, immutable collection enclosed in parentheses ()

# Creating tuples
coordinates = (10, 20)
colors = ("red", "green", "blue")
person = ("Sokha", 20, "Computer Science")
single = (42,)     # ← comma is required for single-item tuple!
empty = ()

print(coordinates)    # (10, 20)
print(colors)         # ('red', 'green', 'blue')
print(len(colors))    # 3

⚠️ Single-Item Tuple Trap

(42) is just the number 42 in parentheses!

(42,) is a tuple with one item — the comma makes it a tuple!

Accessing Tuple Items

colors = ("red", "green", "blue", "yellow", "purple")

# Indexing — same as lists
print(colors[0])      # "red"
print(colors[-1])     # "purple"
print(colors[1:3])    # ('green', 'blue')

# ✅ You CAN read items
print(colors[2])      # "blue"

# ❌ You CANNOT change items
colors[0] = "pink"    # TypeError: 'tuple' object does not support item assignment

Tuples are Immutable — Why Use Them?

# Tuple unpacking — assign multiple values at once!
person = ("Sokha", 20, "Computer Science")

name, age, major = person    # Unpack into 3 variables
print(name)     # "Sokha"
print(age)      # 20
print(major)    # "Computer Science"
# Swap two variables using tuple unpacking
a = 10
b = 20
a, b = b, a    # Swap!
print(a, b)    # 20 10

When to use tuples instead of lists:

  • Data that should NOT change (coordinates, RGB colors, dates)
  • Returning multiple values from a function
  • Dictionary keys (lists cannot be keys, tuples can!)
  • When you want to signal “this data is fixed”

💡 Simple Rule

If data should change → use a list

If data should stay fixed → use a tuple

Tuple Methods & Operations

colors = ("red", "green", "blue", "green", "red", "red")

# count() — how many times?
print(colors.count("red"))     # 3

# index() — first position?
print(colors.index("green"))   # 1

# Membership test
print("blue" in colors)        # True

# Length
print(len(colors))             # 6

# Convert between list and tuple
my_list = list(colors)         # Tuple → List (now mutable!)
my_tuple = tuple(my_list)      # List → Tuple (now immutable!)

Part 5: Dictionaries — Key-Value Pairs

What is a Dictionary?

A dictionary stores data as key: value pairs inside curly braces {}

# Creating a dictionary
student = {
    "name": "Sokha",
    "age": 20,
    "major": "Computer Science",
    "gpa": 3.75
}

print(student)
# {'name': 'Sokha', 'age': 20, 'major': 'Computer Science', 'gpa': 3.75}
# Empty dictionary
empty = {}

# Dictionary with different value types
profile = {
    "username": "sokha_dev",
    "followers": 150,
    "is_active": True,
    "skills": ["Python", "HTML", "CSS"]   # Value can be a list!
}

💡 Key Facts About Dictionaries

  • Unordered (Python 3.7+ keeps insertion order, but don’t rely on it for logic)
  • Mutable — you can add, change, and delete entries
  • Keys must be unique — no duplicate keys allowed
  • Keys must be immutable — strings, numbers, tuples can be keys (NOT lists!)
  • Values can be anything — strings, numbers, lists, other dicts…

Accessing Dictionary Values

student = {
    "name": "Sokha",
    "age": 20,
    "major": "Computer Science",
    "gpa": 3.75
}

# Method 1: Square brackets (raises KeyError if key doesn't exist)
print(student["name"])     # "Sokha"
print(student["gpa"])      # 3.75
# print(student["email"])  # ❌ KeyError: 'email'

# Method 2: .get() (returns None or default if key doesn't exist)
print(student.get("name"))         # "Sokha"
print(student.get("email"))        # None  (no error!)
print(student.get("email", "N/A")) # "N/A" (custom default)

💡 Use .get() to Be Safe

dict["key"] crashes if the key is missing. dict.get("key") returns None safely. Use .get() when you’re not sure if the key exists!

Adding & Updating Dictionary Entries

student = {"name": "Sokha", "age": 20}

# Add a new key-value pair
student["major"] = "Computer Science"
student["gpa"] = 3.75
print(student)
# {'name': 'Sokha', 'age': 20, 'major': 'Computer Science', 'gpa': 3.75}

# Update an existing value
student["age"] = 21
print(student["age"])    # 21

# Update multiple values at once
student.update({"gpa": 3.80, "year": 3})
print(student)
# {'name': 'Sokha', 'age': 21, 'major': 'Computer Science', 'gpa': 3.8, 'year': 3}

Removing Dictionary Entries

student = {
    "name": "Sokha",
    "age": 20,
    "major": "CS",
    "gpa": 3.75,
    "temp": "delete me"
}

# pop() — remove and return a value
removed = student.pop("temp")
print(removed)       # "delete me"
print(student)       # temp is gone

# pop() with default (no error if key missing)
result = student.pop("email", "Not found")
print(result)        # "Not found"

# del — delete by key
del student["gpa"]
print(student)       # gpa is gone

# clear() — remove everything
# student.clear()    # {}

Checking if a Key Exists

student = {"name": "Sokha", "age": 20, "major": "CS"}

# Check with 'in' (checks KEYS, not values)
print("name" in student)       # True
print("email" in student)      # False
print("Sokha" in student)      # False  (checks keys, not values!)

# Safe pattern: check before accessing
if "email" in student:
    print(student["email"])
else:
    print("No email on file")

# Or use .get()
email = student.get("email", "No email on file")
print(email)    # "No email on file"

Part 6: Dictionary Methods & Looping

Looping Through Dictionaries

student = {"name": "Sokha", "age": 20, "major": "CS", "gpa": 3.75}

# Loop through KEYS (default)
for key in student:
    print(key)
# name, age, major, gpa

# Loop through VALUES
for value in student.values():
    print(value)
# Sokha, 20, CS, 3.75

# Loop through KEY-VALUE pairs
for key, value in student.items():
    print(f"{key}: {value}")
# name: Sokha
# age: 20
# major: CS
# gpa: 3.75

🎯 The Three Ways to Loop

  • for key in dict: or for key in dict.keys(): → get keys
  • for value in dict.values(): → get values
  • for key, value in dict.items(): → get both (most common!)

Dictionary .keys(), .values(), .items()

student = {"name": "Sokha", "age": 20, "major": "CS"}

# Get all keys as a list
keys = list(student.keys())
print(keys)      # ['name', 'age', 'major']

# Get all values as a list
values = list(student.values())
print(values)    # ['Sokha', 20, 'CS']

# Get all pairs as a list of tuples
items = list(student.items())
print(items)     # [('name', 'Sokha'), ('age', 20), ('major', 'CS')]

Useful Dictionary Patterns

# Pattern 1: Counting occurrences
sentence = "the cat sat on the mat the cat"
word_count = {}
for word in sentence.split():
    if word in word_count:
        word_count[word] += 1
    else:
        word_count[word] = 1
print(word_count)
# {'the': 3, 'cat': 2, 'sat': 1, 'on': 1, 'mat': 1}
# Pattern 2: Counting with .get() (shorter!)
word_count = {}
for word in sentence.split():
    word_count[word] = word_count.get(word, 0) + 1
print(word_count)
# Same result: {'the': 3, 'cat': 2, 'sat': 1, 'on': 1, 'mat': 1}
# Pattern 3: Grouping items
scores = [("Sokha", 92), ("Dara", 78), ("Sokha", 88), ("Dara", 85)]
grouped = {}
for name, score in scores:
    if name not in grouped:
        grouped[name] = []
    grouped[name].append(score)
print(grouped)    # {'Sokha': [92, 88], 'Dara': [78, 85]}

text = input("Enter a sentence: ").lower()
words = text.split()

# Count each word
frequency = {}
for word in words:
    frequency[word] = frequency.get(word, 0) + 1

Comparison: List vs Tuple vs Dictionary

Feature List [] Tuple () Dictionary {}
Ordered ✅ Yes ✅ Yes ✅ (insertion order)
Mutable ✅ Yes ❌ No ✅ Yes
Duplicates ✅ Allowed ✅ Allowed ❌ Keys unique
Access by Index [0] Index [0] Key ["name"]
Best for Collections that change Fixed data Key-value lookups
Example [85, 92, 78] (10, 20) {"name": "Sokha"}

🎯 When to Use Each

  • List → A collection of similar items (student names, scores, items)
  • Tuple → Fixed data that shouldn’t change (coordinates, RGB colors, dates)
  • Dictionary → Data with labels/names (student profile, settings, config)

Summary: What You’ve Learned

Lists:

  • Create: fruits = ["apple", "banana"]
  • Access: fruits[0], fruits[-1], fruits[1:3]
  • Modify: append(), insert(), remove(), pop(), sort()
  • Loop: for item in list:, for i, item in enumerate(list):

Tuples:

  • Create: point = (10, 20)
  • Access: point[0], point[1]
  • Unpack: x, y = point
  • Immutable — cannot change after creation

Dictionaries:

  • Create: student = {"name": "Sokha", "age": 20}
  • Access: student["name"] or student.get("name")
  • Modify: student["age"] = 21, student.update({...})
  • Loop: for key, value in dict.items():

🚀 Key Takeaways

  • Lists are your go-to for ordered collections — use append() and for loops constantly
  • Tuples are for fixed data — use unpacking to make code cleaner
  • Dictionaries are for labeled data — use .get() for safe access
  • Nested structures (list of dicts) are how real programs organize data
  • Always use in to check membership before accessing: if key in dict:

Common Mistakes to Avoid

# ❌ Mistake 1: Modifying a list while looping over it
numbers = [1, 2, 3, 4, 5]
for n in numbers:
    if n % 2 == 0:
        numbers.remove(n)    # ❌ Skips items!
# ✅ Use a new list instead:
numbers = [n for n in numbers if n % 2 != 0]

# ❌ Mistake 2: Using dict["key"] without checking
student = {"name": "Sokha"}
print(student["email"])          # ❌ KeyError!
print(student.get("email", "N/A"))  # ✅ Safe!

# ❌ Mistake 3: Confusing append and extend
items = [1, 2, 3]
items.append([4, 5])     # ❌ [1, 2, 3, [4, 5]]
items.extend([4, 5])     # ✅ [1, 2, 3, 4, 5]

# ❌ Mistake 4: Forgetting the comma in single-item tuple
t = (42)     # ❌ This is just the integer 42!
t = (42,)    # ✅ This is a tuple!

# ❌ Mistake 5: Using a list as a dictionary key
d = {[1, 2]: "hello"}    # ❌ TypeError! Lists are mutable
d = {(1, 2): "hello"}    # ✅ Tuples can be keys!
